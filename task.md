# Задание

## Разработать Django-приложение с функционалом:

1. CRUD мест проведения мероприятий с данными:
• Название места
• Гео-координаты места (точка)

Разграничение прав доступа:
• Доступно только суперпользователю

2. CRUD мероприятий с данными:
• Название
• Изображения (можно загрузить несколько)
• Превью-изображение (автоматически уменьшать до 200px по наименьшей стороне)
• Описание
• Дата и время публикации
• Дата и время начала проведения
• Дата и время завершения проведения
• Автор
• Место проведения
    - Название
    - Гео-координаты места (точка)
    - Погода в месте проведения
• Рейтинг (от 0 до 25)
• Статус

Разграничение прав доступа:
• Создание/редактирование/удаление - только суперпользователь
• Просмотр - суперпользователю доступны все мероприятия, остальным пользователям только опубликованные мероприятия

Дополнительные параметры:
• Пагинация на странице списка
• Сортировка:
    • Название в алфавитном порядке
    • Дата и время начала проведения
    • Дата и время завершения проведения
• Фильтрация:
    • Диапазон даты и времени начала проведения мероприятия (от и до)
    • Диапазон даты и времени завершения проведения мероприятия (от и до)
    • Место проведения (множественный выбор)
    • Рейтинг мероприятия (от и до)
• Поиск по названию места проведения или мероприятия

3. Выполнение периодических задач:
• Отправка email при публикации мероприятия, с настройками:
    • Список адресатов
    • Тема сообщения
    • Текст сообщения
• Публикация мероприятий при наступлении даты и времени публикации
• Получение погоды в местах проведения (провайдера погоды выбрать самостоятельно, можно реализовать сервис в другом приложении)

Данные погоды:
• Температура по шкале Цельсия
• Влажность воздуха, в %
• Атмосферное давление, в мм ртутного столба
• Направление ветра
• Скорость ветра, в м/с
• Данные погоды должны быть реалистичными и разными (но не обязательно
настоящими)
• Данные погоды должны сохраняться без возможности редактирования

4. Импорт мероприятий из xlsx-файла (openpyxl) с данными:
• Название
• Описание
• Дата и время публикации
• Дата и время начала проведения
• Дата и время завершения проведения
• Название места проведения
• Гео-координаты места
• Рейтинг (от 0 до 25)

5. Экспорт мероприятий в xlsx-файл с возможностью фильтрации по полям:
• Дата и время публикации
• Дата и время начала проведения
• Дата и время завершения проведения
• Место проведения
• Рейтинг

Дополнительные баллы можно набрать за:
• Документирование моделей данных (ERD)
• Покрытие тестами
• Разворачивание приложения в docker
• Адекватную историю коммитов
• Readme с описанием реализованного функционала и инструкциями по разворачиванию локально (и в docker)
